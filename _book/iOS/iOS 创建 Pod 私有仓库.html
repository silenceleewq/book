iOS 创建 Pod 私有仓库

创建 pod lib

pod lib create LibName

放置代码

将代码放在 SLPodCode 下面, 如果没有 Assets 和 Classes 文件件, 可以去 Finder 中拖进来, 只是没有显示在 xcode 里面. 这里的代码就是你私有库中要放置的代码,比如我这里放置了自己自定义的 LRQRedView

修改完后, 使用 pod lib lint 来检查一下 lib 是否有错误. 如果有错误, 对错误进行修改

github 上创建一个私有库来专门存放这个代码库.

在 github 上创建完私有库后. 把上面创建的 lib 推到私有库上面. 推送成功后,再执行:

$ git tag -m 'tag 0.1.0' 0.1.0
$ git push --tags

本地测试 podspec 文件

本地测试 podspec 文件有两种方式:

platform :ios, '9.0'

target 'LRQTestPod' do
  # Uncomment the next line if you're using Swift or would like to use dynamic frameworks
  # use_frameworks!
  # Pods for LRQTestPod
#  pod 'SLPodCode', :path => '/Users/lirenqiang1/yjoy800/Pod/SLPodCode'
  pod 'SLPodCode', :podspec => '/Users/lirenqiang1/yjoy800/Pod/SLPodCode/SLPodCode.podspec'
end

添加完后, 进行 pod install

向 repo 里面添加代码库.

这个repo是专门用来管理 podspec 文件的, 所以: 创建好代码库, 写好 podspec 文件.

pod lib lint

没有问题后, 将 podspec 上传到 repo里面.

pod repo push SLPodRepo SLPodCode.podspec
