iOS IGListKit-创建第一个list

iOS IGListKit-创建第一个list

创建第一个 list

创建 Section Controller

创建一个 Section Controller, 继承 IGListSectionController, 并且至少实现了 cellForItemAtIndex: 和 sizeForItemAtIndex:.

例如在 demo(IGListKit 的官方demo) 中, LabelSectionController 中处理了一个 String 和 配置了只有一个 UILabel 的 cell.

class LabelSectionController: ListSectionController {
  override func sizeForItem(at index: Int) -> CGSize {
    return CGSize(width: collectionContext!.containerSize.width, height: 55)
  }

  override func cellForItem(at index: Int) -> UICollectionViewCell {
    return collectionContext!.dequeueReusableCell(of: MyCell.self, for: self, at: index)
  }
}

创建 UI

在创建了至少一个 section controller 之后, 你必须创建一个 UICollectionView 和 IGListAdapter.

let layout = UICollectionViewFlowLayout()
let collectionView = UICollectionView(frame: .zero, collectionViewLayout: layout)

let updater = ListAdapterUpdater()
let adapter = ListAdapter(updater: updater, viewController: self)
adapter.collectionView = collectionView

这个例子使用了 UIViewController 和普通的 UICollectionViewFlowLayout, IGListAdapterUpdater 来完成的, 如果你有更高级的功能, 可以使用你自己的 layout 和 updater.

关联数据源

最后一步是 IGListAdapter 的数据源并且返回一些数据.

func objects(for listAdapter: ListAdapter) -> [ListDiffable] {
  // this can be anything!
  return [ "Foo", "Bar", 42, "Biz" ]
}

func listAdapter(_ listAdapter: ListAdapter, sectionControllerFor object: Any) -> ListSectionController {
  if object is String {
    return LabelSectionController()
  } else {
    return NumberSectionController()
  }
}

func emptyView(for listAdapter: ListAdapter) -> UIView? {
  return nil
}

创建完 DataSource 后, 需要通过设置 IGListAdapter 的 dataSource 属性来进行关联.

adapter.dataSource = self

你可以返回任何类型的数据,只要这些数据遵守了 IGListDiffable.

Immutability

data 应该是不可变的, 如果是可变的话, 你可能稍后会修改data, 这样 IGListKit 就无法精确的对模型进行 diff 算法了. 所以, 你可以每次都返回一个新的不可变的并且遵守了 IGListDiffable 协议的对象.

Diffing

Model 都需要 conform IGListDiffable 这个协议, 并且实现 diffIdentifier() 和 isEqual(toDiffableObject:)

注意, 对象的 diffIdentifier() 是不能改变的, 如果一个 diffIdentifier() 改变了, 那么 IGListKit 的行为将会不可控.

例如下面这个模型:

class User {
  let primaryKey: Int
  let name: String
  // implementation, etc
}

用户的 primaryKey 是用户的唯一标识, 而 name 不是. 因为后台可能给你这样一个数据:

let shayne = User(primaryKey: 2, name: "Shayne")

过了一段时间, 后台又给了这样一个数据:

let ann = User(primaryKey: 2, name: "Ann")

尽管两个数据表示的是同一个用户, 因为这两个数据有着相同的 primaryKey ,但是这两个数据是不同的, 因为用户的 name 改变了.

要想表达这样的关系, 就需要像下面这样写:

extension User: ListDiffable {
  func diffIdentifier() -> NSObjectProtocol {
    return primaryKey
  }

  func isEqual(toDiffableObject object: Any?) -> Bool {
    if let object = object as? User {
      return name == object.name
    }
    return false
  }
}

这样的话, 如果两个对象的 primaryKey 和 name 相同的话, IGListKit 就不会去刷新这条数据, 这样就避免了不必要的 UI 更新, 尽管他们是不同的对象.

如果你想要去刷新某些 cell 的话,就要在 isEqual(toDiffableObject:) 中返回 false.

高级功能 (Advanced Features)

Working Rangew

Working Range 就是 Section Controller 当前还没看见的, 但是在屏幕附近的范围. 当这些范围即将出现或者消失的时候, Section Controller 就会收到通知. 这个概念可以让 Section Controller 提前为那些即将出现的范围进行内容上的准备.

要用这个功能, 必须在 IGListAdapter 初始化的时候赋值才能工作. 这个值是 cell 的宽(水平滚动)或高(纵向滚动)的倍数. let adapter = ListAdapter(updater: ListAdapterUpdater(), viewController: self, workingRangeSize: 1) // 1 before/after visible objects

你可以设置 section controller 的代理 workingRangeDelegate, 来接收事件.

Supplementary Views.

设置 section controller 的代理 supplementaryViewSource, 并且实现 IGListSupplementaryViewSource 协议, 这个协议的工作方式和返回一个定制过的cell一样.

Display Delegate

设置 section controller 的代理 displayDelegate, 来接收 section controller 和 cell 的显示事件.

Custom Updaters

默认的 IGListAdapterUpdater 应该处理 UICollectionView 的任何更新, 如果你有些一些特别的更新, 创建一个对象, 并且遵守了 IGListUpdatingDelegate 协议, 然后用这个对象初始化一个新的 IGListAdapter
